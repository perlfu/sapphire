RVM : Editing JikesRVM in an IDE
This page last changed on May 23, 2007 by rgarner.

One goal of the JikesRVM project over recent years has been the ability to develop JikesRVM in a development environment such as Eclipse.  This has been possible for the MMTk component since 2005, and as of early 2007 (release 2.9.0) it is possible to work with the majority of the JikesRVM codebase in Eclipse and similar environments.  This is not yet as straightforward as we would like, and can be expected to improve with time.

Editing JikesRVM in Eclipse

These instructions assume you are working on 32-bit ia32, but should work for other architectures/address lengths by making the obvious substitutions. 

  1. Create a JikesRVM source tree either via SVN checkout or untar-ing a distribution.

    $ svn co https://jikesrvm.svn.sf.net/svnroot/jikesrvm/rvmroot/trunk jikesrvm


  2. From the top-level directory you just created, run the ant task prepare-source with arbitrary (but valid) host name and config.

    $ cd jikesrvm
    $ ant -Dhost.name=ia32-linux -Dconfig.name=prototype-opt


  3. Create an Eclipse Java project.  Give it a name, and select 'Create project from existing source', with Directory the main jikesrvm directory (ie the one created by the svn checkout). Click 'Next>'
  4. Under the 'source' tab, remove the following source directories that Eclipse automatically identified:

    components/classpath/93/classpath (and all subdirectories)
    generated/ppc-*
    generated/vmmagic/64
    rvm/src-generated/*
    target/*
    testing/*


  5. Still on the same screen, exclude the architecture-specific packages under rvm/src

    org.jikesrvm.**.ppc (or ia32)
    org.jikesrvm.tools.header_gen


  6. On the 'Libraries' tab of the same screen, remove all libraries except classpath.jar. This includes removing the JDK libraries, which conflict with Classpath.
  7. Click 'Finish'.
    There are other ways to configure Eclipse to edit JikesRVM, but at time of writing this is the most straightforward.
